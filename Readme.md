### What it is

TMOS13 is a protocol-driven AI platform where professionals deploy AI
representatives through structured behavioral contracts called **packs**.
The conversation is the product вЂ” deliverables (intake forms, candidate
screenings, case files, analyses, pilgrimage records) are produced through
governed conversation, not chat wrappers.

**The core inversion:** most AI frameworks give the model autonomy and tell
it to use tools wisely. TMOS13 gives the protocol authority and tells the
model to express it faithfully. The pack decides what happens. The model
decides how to say it.

---

### How it works

**Write a pack** вЂ” a plain text manifest defining an AI session's identity,
routing logic, conversation flow, formatting rules, and deliverable structure.

**Configure a manifest** вЂ” a JSON file defining the session's behavioral
parameters: cartridges (modules), commands, features, state schema, privacy.

**The engine handles everything else** вЂ” session state, deterministic routing,
prompt assembly, vault persistence, cross-session memory, multi-channel delivery.

Swap the pack, change the product. The engine never changes.

### Pack format

```yaml
# header.yaml
pack_id: your_pack_id
name: Your Pack Name
category: vertical
status: active
version: 1.0.0
estimated_turns: 8-14
description: One sentence. What does this session do?
deliverable: what_it_produces
```

```markdown
# YOUR PACK вЂ” GOVERNING PROTOCOL

## Identity
You are [role]. [Two sentences on what that means.]

## Session Structure
[Phases, turn counts, routing logic.]

## Voice
[Tone, things never to say, what makes this voice distinct.]

## Deliverable
[What gets produced at session end. Format. Fields.]
```

That's the format. No SDK. No deployment infrastructure required.
Write the protocol. The engine runs it.
